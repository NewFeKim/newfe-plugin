# 슬라이드 레이아웃 추천기 (Slide Layout Picker)

슬라이드 콘텐츠의 성격과 전달 목적을 입력받아 10가지 레이아웃 패턴 중
가장 적합한 1순위와 대안 1~2개를 디자인 원칙 근거와 함께 추천한다.
visual-technique-picker와 짝을 이뤄 "기법 선택 → 슬라이드 배치"를 완성한다.

## 동작 순서

### 1단계: 입력 파악

사용자 메시지에서 아래 5가지를 추출한다. 명시되지 않은 항목은 합리적으로
가정하되, **슬라이드에 담을 핵심 내용**이 불명확하면 그 한 가지만 되묻는다.

- **핵심 내용** (필수): 텍스트 중심 / 이미지·비주얼 중심 / 수치·데이터 / 비교·대조 / 프로세스·단계 / 한 문장 메시지
- **텍스트량**: 제목+한 줄 / 짧은 단락(3~5줄) / 긴 본문(6줄+) / 불릿 리스트
- **비주얼 비중**: 없음 / 보조(30% 이하) / 절반(50%) / 주인공(70%+)
- **비교 대상 수**: 없음 / 2개 / 3개 / 4개+
- **발표 목적**: 설득·제안 / 정보 전달 / 보고·결과 공유 / 교육·강의 / 인스피레이션

### 2단계: 콘텐츠 성격 → 레이아웃 매핑

"핵심 내용"을 기준으로 후보 레이아웃을 좁힌다. 가장 강한 신호다.

| 핵심 내용 | 1순위 레이아웃 | 보조 후보 |
|-----------|--------------|-----------|
| **한 문장 메시지·인용** | 센터드 타이포 | 풀스크린 이미지+오버레이 |
| **핵심 수치 1~2개** | 빅넘버 | 센터드 타이포 |
| **데이터·차트** | 차트 포커스 | 좌우 분할(설명+차트) |
| **2개 비교** | 좌우 비교 | 표 레이아웃 |
| **3~4개 비교·옵션** | 그리드(2×2 또는 1×3) | 표 레이아웃 |
| **단계·프로세스** | 타임라인·스텝 | 번호형 불릿 |
| **텍스트+이미지 균형** | 좌우 분할 | T자형(상단 이미지+하단 텍스트) |
| **이미지가 주인공** | 풀스크린 이미지 | T자형 |
| **긴 설명·강의 내용** | 좌측 타이틀+우측 본문 | 번호형 불릿 |
| **인스피레이션·오프닝** | 풀스크린 이미지+오버레이 | 센터드 타이포 |

### 3단계: 발표 목적·텍스트량으로 보정

후보 중에서 아래 기준으로 최종 선택한다.

- **설득·제안**: 임팩트 우선 → 빅넘버, 센터드 타이포, 좌우 비교 선호
- **정보 전달**: 명료함 우선 → 타임라인, 그리드, 좌우 분할 선호
- **보고·결과**: 근거 우선 → 차트 포커스, 표 레이아웃 선호
- **교육·강의**: 가독성 우선 → 좌측 타이틀+본문, 번호형 불릿 선호
- **텍스트 6줄 이상**: 슬라이드 분리를 먼저 권고 → 불가피하면 글씨 최소화 레이아웃
- **비주얼 없음 + 긴 텍스트**: 레이아웃보다 슬라이드 쪼개기가 정답임을 먼저 알린다

### 4단계: 한 슬라이드 한 메시지 원칙 체크

추천 전에 아래 원칙에 비춰 콘텐츠를 점검하고, 위반 시 한 줄로 먼저 경고한다.

| 원칙 | 체크 기준 | 위반 시 조치 |
|------|----------|-------------|
| **한 슬라이드 한 메시지** | 핵심 내용이 2개 이상인가? | 슬라이드 분리 권고 |
| **시선 흐름 (Z·F·센터)** | 시선이 자연스럽게 흐르는가? | 레이아웃 방향 제안 |
| **텍스트 6줄 한도** | 불릿 6개 초과인가? | 축약 또는 분리 권고 |
| **여백 최소 20%** | 요소가 슬라이드를 꽉 채우는가? | 여백 확보 방법 제안 |
| **폰트 사이즈 24pt+** | 뒤에서 읽힐 발표인가? | 텍스트 축소 경고 |

### 5단계: visual-technique-picker 연계

직전에 visual-technique-picker를 사용한 경우, 또는 사용자가 기법(차트/
다이어그램/타임라인 등)을 이미 언급한 경우, 그 기법에 맞는 레이아웃을
우선 매핑한다.

| visual-technique-picker 추천 기법 | 자연스러운 슬라이드 레이아웃 |
|-----------------------------------|-----------------------------|
| 차트 | 차트 포커스 / 좌우 분할(설명+차트) |
| 다이어그램 | 풀스크린(다이어그램) / 좌우 분할 |
| 플로우차트 | 풀스크린 / 타임라인·스텝 |
| 타임라인 | 타임라인·스텝 레이아웃 |
| 인포그래픽 | 풀스크린 / 그리드 |
| 비포애프터 | 좌우 비교 레이아웃 |
| 비주얼 메타포 | 풀스크린 이미지+오버레이 / 센터드 타이포 |
| 로드맵 | 타임라인·스텝 / 그리드 |
| 빅 수치 강조 | 빅넘버 레이아웃 |

### 6단계: 출력

아래 형식으로 답한다. 레이아웃은 텍스트 구조로 시각화한다.

```
📐 추천 레이아웃: [레이아웃 이름]

[레이아웃 구조도 — ASCII로 간단히]
┌─────────────────────────────┐
│  제목 영역                  │
│                             │
│  왼쪽 콘텐츠 │ 오른쪽 콘텐츠 │
│             │               │
└─────────────────────────────┘

배치 원칙: [왜 이 레이아웃인지, 어떻게 배치하는지 1~2문장]
시선 흐름: [Z형 / F형 / 센터 집중 — 어디서 시작해 어디로 가는지]

⚠️ 체크: [한 슬라이드 한 메시지 원칙 위반 여부 또는 텍스트 과다 경고]

🔄 대안 레이아웃:
- [대안1] — [언제 이게 더 나은지]
- [대안2] — [언제 이게 더 나은지]

💡 팁: [이 레이아웃의 흔한 실수 또는 한 단계 업그레이드 방법]
```

추가 질문이 없으면 바로 추천을 내고, 끝에 "이 레이아웃에 맞는 색상 팔레트도
잡아드릴까요? (color-palette-picker)"를 한 번만 제안한다.

## 10가지 레이아웃 레퍼런스

판단 시 아래 표를 근거로 삼는다.

| 레이아웃 | 구조 | 강점 | 약점 | 적합한 상황 |
|---------|------|------|------|------------|
| **센터드 타이포** | 중앙 대형 텍스트 | 임팩트·기억 | 정보량 제한 | 인용구, 오프닝, 핵심 한 줄 |
| **빅넘버** | 대형 숫자 + 짧은 레이블 | 수치 임팩트 | 수치 없으면 불가 | KPI, 성과 수치 강조 |
| **풀스크린 이미지** | 이미지 전체 + (옵션) 오버레이 텍스트 | 감성·몰입 | 이미지 품질 의존 | 오프닝, 섹션 구분, 분위기 전환 |
| **좌우 분할** | 왼쪽 텍스트·설명 / 오른쪽 비주얼·데이터 | 균형·설명력 | 공간 각각 좁아짐 | 기능 설명, 데이터+해석, 인물+인용 |
| **좌우 비교** | 왼쪽 A / 오른쪽 B (대칭) | 비교 명료 | 2개 고정 | 비포애프터, 장단점, 선택지 2개 |
| **그리드 (2×2)** | 4분할 동등 칸 | 다항목 정리 | 3개 이상 비교 | 4가지 특징, 옵션 비교, 사분면 |
| **타임라인·스텝** | 수평 또는 수직 시간축 | 순서·흐름 명확 | 단계 5개 초과 시 복잡 | 프로세스, 역사, 로드맵 |
| **차트 포커스** | 차트 70%+ / 제목+한줄 요약 | 데이터 중심 | 설명 공간 부족 | 데이터 발표, 분석 슬라이드 |
| **T자형** | 상단 이미지 / 하단 텍스트 | 이미지+설명 공존 | 이미지 가로 제약 | 제품 소개, 사례 슬라이드 |
| **번호형 불릿** | 제목 + 번호 붙은 리스트 | 순서 강조 | 텍스트 과다 위험 | 3~5가지 근거, 순서 있는 설명 |

## 예시

**입력**: "분기 매출이 30% 성장했다는 걸 임원한테 발표하고 싶어. 숫자 하나만 강조"

→ 핵심 내용=수치 1개, 텍스트량=최소, 목적=설득·보고

```
📐 추천 레이아웃: 빅넘버 (Big Number)

┌─────────────────────────────┐
│                             │
│        +30%                 │
│   (초대형, 화면 중앙)        │
│                             │
│   Q3 매출 전년 동기 대비     │
│   (작은 서브텍스트, 하단)    │
└─────────────────────────────┘

배치 원칙: 숫자 하나를 슬라이드 전체로 키워 임원이 한눈에 인식하게 합니다.
           서브텍스트는 최소화해 "30%"만 기억에 남기는 구조입니다.
시선 흐름: 센터 집중 — 숫자로 바로 향한 뒤 하단 레이블로 내려감

🔄 대안 레이아웃:
- 센터드 타이포 — 숫자보다 "역대 최고 분기"처럼 문장 메시지가 중심일 때
- 차트 포커스 — 분기별 추세(5분기 이상)를 함께 보여줘야 할 때

💡 팁: 숫자 색상을 브랜드 포인트 색으로 쓰고 배경은 흰색으로 유지하면
       "깔끔한 자신감" 인상을 줍니다. 숫자 옆에 작은 화살표(↑) 아이콘 하나만
       더해도 방향성이 명확해집니다.
```

**입력**: "우리 서비스 기존 방식 vs 새 방식 차이를 슬라이드로 보여주고 싶어.
visual-technique-picker에서 비포애프터 추천받았어."

→ 기법=비포애프터, 핵심 내용=2개 비교, 목적=설득

```
📐 추천 레이아웃: 좌우 비교 (Side-by-side Comparison)

┌─────────────────────────────┐
│  기존 방식      │  새 방식   │
│  ─────────      │  ──────── │
│  • 단점 A       │  ✓ 개선 A │
│  • 단점 B       │  ✓ 개선 B │
│  • 단점 C       │  ✓ 개선 C │
│                 │           │
│  [이미지/아이콘] │ [이미지]  │
└─────────────────────────────┘

배치 원칙: 좌(Before) / 우(After) 대칭 구조로 시선이 자연스럽게 비교합니다.
           왼쪽에 문제, 오른쪽에 해결을 배치하면 "왜 바꾸는가"가 즉시 전달됩니다.
시선 흐름: Z형 — 왼쪽 위 → 오른쪽 위 → 왼쪽 아래 → 오른쪽 아래

🔄 대안 레이아웃:
- 그리드(2×2) — 비교 포인트가 4개 이상이고 각각을 독립적으로 부각할 때
- 타임라인·스텝 — "이전 → 이후"의 시간 흐름을 강조하고 싶을 때

💡 팁: 새 방식 칸에 브랜드 포인트 색을 살짝 깔거나(연한 배경) 체크마크(✓)를
       강조색으로 표시하면, 시선이 자연스럽게 오른쪽(새 방식)으로 끌립니다.
       양쪽에 같은 수의 항목을 넣어야 대칭이 무너지지 않습니다.
```
