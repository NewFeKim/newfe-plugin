# 프레젠테이션 플래너 (Presentation Planner)

10가지 기준을 입력받아 슬라이드별 콘텐츠·레이아웃·디자인 시스템을 설계하고,
확인 후 실제 .pptx 파일 생성까지 연결한다.

---

## 1단계: 10가지 기준 수집

아래 항목을 사용자 메시지에서 추출한다. 명시되지 않은 항목은 아래 기본값으로
가정하되, **목적·청중·핵심 메시지** 3가지가 불명확하면 한 번에 모아서 묻는다
(항목마다 따로 묻지 말고 한 메시지로).

| 번호 | 기준 | 필수 | 기본값 |
|------|------|------|--------|
| 1 | **목적**: 발표용 / 제출용 / 제안서 / 교육자료 | ✓ | — |
| 2 | **청중**: 임원 / 고객 / 투자자 / 학습자 | ✓ | — |
| 3 | **핵심 메시지**: 발표 후 남길 한 문장 | ✓ | — |
| 4 | **스토리라인**: 문제→해결 / 개념→실습 / Before→After / 데이터 스토리텔링 / 타임라인·로드맵 | | 목적에서 유추 |
| 5 | **레퍼런스**: 스크린샷 / 기존 PPT / 브랜드 자료 | | 없음 |
| 6 | **서식**: 비율(16:9/4:3), 제목 위치, 본문 줄 수, 여백 | | 16:9, 상단, 5줄, 표준 |
| 7 | **컬러**: 메인/배경/텍스트/강조 | | 디자인 스타일에서 도출 |
| 8 | **폰트**: 제목/본문/숫자용 | | 스타일별 기본 세트 |
| 9 | **디자인 스타일**: B2B SaaS / 교육자료 / 보고서 / IR / 컨설팅 / 스타트업 브리핑 | | 목적+청중에서 유추 |
| 10 | **금기사항**: 과한 그라데이션 금지, 의미 없는 아이콘 금지, 제목 밑 가로선 금지 등 | | 스타일별 기본 금기 적용 |

---

## 2단계: 덱 설계 전략 도출

수집한 기준을 아래 매핑표로 설계 방향을 결정한다.

### 목적 × 청중 → 톤·밀도

| 목적 \ 청중 | 임원 | 고객 | 투자자 | 학습자 |
|------------|------|------|--------|--------|
| **발표용** | 수치 중심, 간결, 3초 룰 | 혜택 중심, 공감 유도 | 성장 스토리, 숫자 근거 | 개념 → 예시 단계 |
| **제출용** | 결론 먼저, 근거 압축 | 비교표, 선택지 명시 | 재무지표 필수 | 체크리스트 형식 |
| **제안서** | ROI 명시, 리스크 최소화 | 문제 공감 → 솔루션 | TAM/SAM/SOM 필수 | 단계별 실습 가이드 |
| **교육자료** | 핵심 3가지로 압축 | 쉬운 언어, 비유 활용 | — | 학습목표 → 정리 |

### 스토리라인 → 슬라이드 시퀀스 템플릿

**문제→해결** (설득, 제안서, IR에 적합)
```
표지 → 목차(선택) → 문제 정의 → 현황/고통 포인트 → 해결책 개요 → 
솔루션 상세 → 효과/증거(데이터) → 도입 방법/로드맵 → CTA/마무리
```

**개념→실습** (교육자료, 강의, 온보딩에 적합)
```
표지 → 학습목표 → 개념 설명 → 핵심 원리 → 예시/사례 → 
실습/활동 → 흔한 실수 → 요약 → Q&A
```

**Before→After** (성과 보고, 리뷰, 비교 제안에 적합)
```
표지 → 배경 → 현재 상황(Before) → 문제점 진단 → 
전환점/솔루션 → 이후 모습(After) → 수치로 증명 → 다음 스텝
```

**데이터 스토리텔링** (분석 보고, KPI 리뷰, 리서치 발표에 적합)
```
표지 → 질문 제기("왜 이 데이터를 보는가") → 전체 트렌드 조망 →
세그먼트 분해 → 이상치·인사이트 포커스 → 원인 가설 → 
액션 아이템 → 다음 측정 시점
```
- 각 슬라이드는 차트/그래프 1개 + 한 줄 인사이트 구조를 기본으로 한다
- 숫자는 반드시 맥락(전년比, 업계 평균, 목표 대비)과 함께 제시한다

**타임라인/로드맵** (프로젝트 계획, 제품 로드맵, 마일스톤 발표에 적합)
```
표지 → 목표 선언 → 현재 위치(지금까지) → 단계별 계획(Phase 1→N) →
각 단계 상세(필요 시) → 리소스·전제조건 → 리스크 및 대응 → 
최종 목표 상태
```
- 슬라이드 1장당 Phase 1개를 기본으로 하되, Phase가 많으면 전체 타임라인을 1장에 압축 후 상세 확장
- 날짜/분기 표기는 슬라이드마다 일관되게 유지한다

### 디자인 스타일 → 컬러·폰트 기본 세트

**B2B SaaS**
- 컬러: 네이비/다크블루 메인(#1E3A8A), 흰 배경, 파란 강조(#3B82F6)
- 폰트: 제목 Inter Bold / 본문 Inter Regular / 숫자 JetBrains Mono
- 특징: 정보 밀도 높음, 아이콘 최소화, 데이터 중심

**교육자료**
- 컬러: 밝은 인디고 메인(#4F46E5), 연한 회색 배경, 앰버 강조(#F59E0B)
- 폰트: 제목 Pretendard Bold / 본문 Pretendard Regular / 숫자 Pretendard SemiBold
- 특징: 여백 넉넉, 아이콘 보조 허용, 단계 번호 강조

**보고서**
- 컬러: 차콜 메인(#1F2937), 흰 배경, 슬레이트 보조(#64748B)
- 폰트: 제목 KoPub Batang Bold / 본문 KoPub Batang / 숫자 Helvetica Neue
- 특징: 표·차트 중심, 텍스트 8pt 이상, 기관 신뢰감

**IR (투자자 발표)**
- 컬러: 딥블랙 배경(#0A0A0A), 흰 텍스트, 골드 강조(#EAB308)
- 폰트: 제목 Montserrat ExtraBold / 본문 Montserrat Regular / 숫자 Oswald
- 특징: 임팩트 우선, 숫자 크게, 슬라이드당 메시지 1개 엄수

**컨설팅/전략 보고서** (McKinsey·BCG 스타일, 경영진 전략 보고, 컨설팅 제안서에 적합)
- 컬러: 흰 배경, 진한 차콜 텍스트(#111827), 강조 딥블루(#1D4ED8), 보조 라이트그레이(#F3F4F6)
- 폰트: 제목 Georgia Bold / 본문 Arial Regular / 숫자 Arial Bold
- 특징: 여백 극대화, 슬라이드 상단에 1줄 핵심 결론(Headline), 차트+주석 조합, 색 최소화(흑백+강조 1색)
- 금기: 배경색 사용 금지, 클립아트·아이콘 금지, 슬라이드 제목이 "개요/현황" 같은 명사형이면 무조건 메시지형으로 교체

**스타트업 브리핑** (팀 공유, 올핸즈, 프로덕트 업데이트에 적합)
- 컬러: 흰 배경, 밝은 바이올렛 메인(#7C3AED), 라임 강조(#84CC16), 연회색 보조(#E5E7EB)
- 폰트: 제목 Pretendard ExtraBold / 본문 Pretendard Regular / 숫자 Pretendard SemiBold
- 특징: 이모지·아이콘 적극 허용, 짧은 문장, 밈·유머 가능, 1슬라이드=1업데이트 원칙
- 슬라이드 수: 10장 이하 권장, 발표자 노트 대신 슬라이드 자체에 정보 압축

## 3단계: 슬라이드별 기획서 출력

아래 형식으로 슬라이드 하나씩 출력한다. 총 슬라이드 수는
발표 시간(분) × 1.5~2장, 제출용은 콘텐츠 기준으로 결정한다.

```
## 슬라이드 [N]: [제목]
**레이아웃**: [slide-layout-picker 기준: 빅넘버 / 좌우분할 / 풀스크린 등]
**핵심 콘텐츠**:
- [불릿 또는 콘텐츠 요소]
**비주얼 제안**: [차트 / 다이어그램 / 이미지 등, visual-technique-picker 연계]
**디자인 노트**: [폰트 크기, 강조색 사용, 특별 주의사항]
```

출력 규칙:
- 슬라이드당 본문 불릿 **5줄 이하** (서식 기준 우선)
- 비주얼이 있으면 텍스트 줄 수를 3줄 이하로 줄인다
- 슬라이드 제목은 **메시지형** (명사 나열 말고 "X가 Y를 해결한다" 형태 권장)

---

## 4단계: 디자인 시스템 출력

슬라이드 기획서 뒤에 아래 디자인 시스템 카드를 붙인다.

```
---
## 🎨 디자인 시스템

**컬러 팔레트**
- 메인:   #XXXXXX  [색 이름]
- 배경:   #XXXXXX
- 텍스트: #XXXXXX
- 강조:   #XXXXXX

**폰트**
- 제목:  [폰트명] [굵기] [크기]pt
- 본문:  [폰트명] [굵기] [크기]pt
- 숫자:  [폰트명] — 데이터·통계 전용

**서식 규칙**
- 비율: [16:9 / 4:3]
- 제목 위치: [상단 좌측 / 상단 중앙]
- 본문 최대 줄 수: [N]줄
- 여백: 상하 [N]px / 좌우 [N]px

**금기사항 체크리스트**
- [ ] 3색 초과 그라데이션 금지
- [ ] 장식용 아이콘(의미 없는 것) 금지
- [ ] 제목 아래 구분선(가로선) 금지
- [ ] [사용자가 추가한 금기사항]
---
```

## 5단계: 파일 생성 제안

기획서 출력 후 아래 한 줄만 덧붙인다:

> "이 기획서로 실제 `.pptx` 파일을 만들어 드릴까요? (python-pptx 기반, 폰트·컬러 적용)"

사용자가 Yes이면 `pptx` 스킬을 호출해 파일 생성으로 이어간다.
사용자가 원하는 슬라이드를 수정하거나 기준을 바꾸고 싶으면 해당 항목만
업데이트하고 영향받는 슬라이드만 다시 출력한다(전체 재출력 금지).

---

## 디자인 원칙 레퍼런스

판단 시 아래 원칙을 근거로 삼는다.

| 원칙 | 설명 | 위반 신호 |
|------|------|----------|
| **한 슬라이드 한 메시지** | 슬라이드 제목이 곧 메시지여야 함 | 불릿 7개 이상, 제목이 "개요" |
| **3초 룰** | 3초 안에 핵심을 읽을 수 있어야 함 | 본문 6줄 초과, 폰트 18pt 미만 |
| **시선 흐름 (Z·F)** | 좌상→우상→좌하→우하(Z) 또는 좌→우(F) | 중요 정보가 우하단에 위치 |
| **여백은 디자인** | 빈 공간이 불안해서 채우지 말 것 | 슬라이드가 꽉 참, 여백 10% 미만 |
| **색은 의미를 가짐** | 강조색은 진짜 강조할 때만 | 페이지마다 강조색이 3곳 이상 |
| **폰트는 2종으로** | 제목/본문 2종이면 충분 | 3종 이상 혼용 |

## 예시

**입력**: "스타트업 투자자 발표 자료 만들어줘. 우리가 물류 SaaS로 배송 지연을
30% 줄였다는 게 핵심이야. 10분 발표."

→ 목적=발표용, 청중=투자자, 핵심 메시지="물류 SaaS로 배송 지연 30% 감소",
   스토리라인=문제→해결, 스타일=IR, 총 슬라이드=15장 내외

출력 예시 (일부):

```
## 슬라이드 1: [회사명] — 배송 지연을 30% 줄이는 물류 SaaS
레이아웃: 풀스크린 이미지 + 오버레이 텍스트
핵심 콘텐츠:
- 회사명 + 한 줄 설명
- "배송 지연 -30%" 빅 서브타이틀
비주얼 제안: 물류 창고 또는 배송 트럭 고화질 이미지 (어두운 오버레이)
디자인 노트: 텍스트는 흰색, 강조 수치는 골드(#EAB308) 적용

## 슬라이드 3: 한국 이커머스, 배송 지연이 이탈의 1위 원인이다
레이아웃: 빅넘버 (좌) + 설명 텍스트 (우)
핵심 콘텐츠:
- 빅넘버: "68%"
- 서브: "배송 지연 경험 후 재구매 포기 비율 (출처: 통계)"
- 우측: 현재 물류 시스템의 3가지 구조적 문제
비주얼 제안: 없음 (숫자가 비주얼)
디자인 노트: 68%는 골드, 나머지 텍스트는 화이트
```

---

**입력**: "신입사원 온보딩 교육자료 만들려고. 우리 회사 개발 프로세스 설명하는 거야."

→ 목적=교육자료, 청중=학습자, 스토리라인=개념→실습, 스타일=교육자료

출력 예시 (일부):

```
## 슬라이드 2: 오늘 배우고 나면 혼자 PR을 올릴 수 있습니다
레이아웃: 센터드 타이포 + 학습목표 리스트
```
